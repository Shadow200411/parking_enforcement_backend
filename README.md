# AI Parking Enforcement Backend

A lightning-fast, asynchronous REST API built with **FastAPI** and **PostgreSQL**. This system acts as the "brain" for an AI-powered police car camera, receiving real-time license plate detections and applying business logic to flag parking violations.

## Features
* **Asynchronous Database:** Powered by SQLAlchemy 2.0 and `asyncpg` for high-throughput AI traffic.
* **Smart Decision Engine:** Automatically filters out legal cars and prevents duplicate spam using an in-memory cache.
* **Fully Dockerized:** Spin up the entire database and web server with a single command.
* **Interactive API Docs:** Built-in Swagger UI for easy frontend testing.

## Getting Started

### Prerequisites
You only need two things installed on your machine to run this project:
1. [Git](https://git-scm.com/)
2. [Docker Desktop](https://www.docker.com/products/docker-desktop/)

### Installation & Setup

**1. Clone the repository**
```bash
git clone [https://github.com/Shadow200411/parking_enforcement_backend.git](https://github.com/Shadow200411/parking_enforcement_backend.git)
cd parking_enforcement_backend 
```
**2. Start the application**
```bash
docker compose up --build
```
Note: The first time you run this, Docker will download PostgreSQL and Python. On subsequent runs, it will boot up in seconds. Alembic migrations are handled automatically on startup!

**3. Test the API**
Once the terminal says Uvicorn running, open your web browser and navigate to the interactive API documentation:
-> http://localhost:8000/docs

Typical Workflow for Testing
Go to the /docs page.

Run POST /api/v1/seed to generate a test parking lot.

Run POST /api/v1/detections with a test payload (e.g., "TEST-123").

Run GET /api/v1/flags to see the resulting violation generated by the Decision Engine.
