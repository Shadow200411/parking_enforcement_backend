# AI Parking Enforcement Backend

A lightning-fast, asynchronous REST API built with **FastAPI** and **PostgreSQL**. This system acts as the "brain" for an AI-powered police car camera, receiving real-time license plate detections and applying business logic to flag parking violations.

## Features
* **Asynchronous Database:** Powered by SQLAlchemy 2.0 and `asyncpg` for high-throughput AI traffic.
* **Smart Decision Engine:** Automatically filters out legal cars and prevents duplicate spam using an in-memory cache.
* **Fully Dockerized:** Spin up the entire database and web server with a single command.
* **Interactive API Docs:** Built-in Swagger UI for easy frontend testing.

## Getting Started

### Prerequisites
You only need two things installed on your machine to run this project:
1. [Git](https://git-scm.com/)
2. [Docker Desktop](https://www.docker.com/products/docker-desktop/)

### Installation & Setup

**1. Clone the repository**
```bash
git clone [https://github.com/Shadow200411/parking_enforcement_backend.git](https://github.com/Shadow200411/parking_enforcement_backend.git)
cd parking_enforcement_backend 
```
**2. Start the application**
```bash
docker compose up --build
```
Note: The first time you run this, Docker will download PostgreSQL and Python. On subsequent runs, it will boot up in seconds. Alembic migrations are handled automatically on startup!

### 3. Seed the database

Once the container is running, populate your database with test parking lots, cars, subscriptions, and user accounts by running the seed script:

```bash
docker compose exec web python -m app.seed
```

## Typical Workflow for Testing

### 1. Fire the camera

Go to `POST /api/v1/detections` and send a test payload, for example:

```json
{
  "car_registration_no": "CD-01-AAA",
  "parking_id": 1,
  "confidence_score": 0.95,
  "evidence_image_url": "/images/CD-01-AAA_evidence.jpg"
}
```

The Decision Engine will process it instantly.

Here is a ready-to-paste JSON file with quick examples for testing the API:
https://drive.google.com/file/d/1IR5DtN1TuTWh9WhNF0gw7AlyHOCJfzdM/view?usp=sharing

Make sure you take one at a time to test. Do not send all the file at once.

### 2. View the flags

Go to `GET /api/v1/flags` and click **Execute** to see the resulting violations generated by the engine.

### 3. Log in (authenticate)

Click the green **Authorize** button at the top right of the page. Enter the admin credentials to securely save your JWT token to your browser.

### 4. Act as an officer

Go to the locked `PATCH /api/v1/flags/{flag_id}/verify` endpoint. Enter the ID of a low-confidence flag and approve or reject it.
